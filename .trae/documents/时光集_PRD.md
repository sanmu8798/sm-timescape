# 【时光集】产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称
**时光集** - 记录旅行足迹，收藏美好时光

### 1.2 产品定位
旅行记录工具，帮助用户记录全国各个城市的景点、美食、风俗特色，通过打卡、评论、打分等功能，打造个人专属的旅行记忆集。

### 1.3 目标用户
- **旅行爱好者**：经常旅游，喜欢记录旅行点滴
- **本地居民**：分享本地特色，展示家乡魅力（后续可扩展上传功能）
- **学生群体**：预算有限的年轻旅行者
- **商务人士**：出差期间探索当地美食

### 1.4 产品愿景
成为用户记录旅行、分享体验、发现美好的首选平台，让每一次旅行都留下珍贵印记。

## 2. 核心功能需求

### 2.1 用户系统

| 功能模块 | 功能描述 | 需求来源 |
| :--- | :--- | :--- |
| 用户注册 | 支持手机号/微信/邮箱注册 | 基础需求 |
| 用户登录 | 支持手机号/微信/邮箱登录，记住密码 | 基础需求 |
| 用户中心 | 个人资料编辑、头像上传、打卡记录查看 | 基础需求 |
| 退出登录 | 安全退出当前账号 | 基础需求 |

### 2.2 城市探索

| 功能模块 | 功能描述 | 需求来源 |
| :--- | :--- | :--- |
| 城市列表 | 展示全国各城市，支持搜索筛选 | 核心需求 |
| 城市详情 | 展示城市简介、景点、美食、风俗 | 核心需求 |
| 位置获取 | 获取用户当前位置，推荐附近城市 | 用户要求 |

### 2.3 打卡系统（核心功能）

| 功能模块 | 功能描述 | 需求来源 |
| :--- | :--- | :--- |
| 地点打卡 | 到达具体地点签到打卡 | 用户要求：地点打卡 |
| 内容打卡 | 完成特定任务打卡（如品尝美食） | 用户要求：内容打卡 |
| 图片上传 | 打卡时需上传图片，类似发动态 | 用户要求 |
| 打卡记录 | 查看个人所有打卡记录 | 基础需求 |

### 2.4 评论评分系统

| 功能模块 | 功能描述 | 需求来源 |
| :--- | :--- | :--- |
| 景点评论 | 对景点进行评论和打分（1-5星） | 用户要求：景点 |
| 美食评论 | 对餐厅/美食进行评论和打分 | 用户要求：美食 |
| 城市评论 | 对整个城市进行评价 | 用户要求：城市 |
| 风俗评论 | 对当地风俗文化进行评价 | 用户要求：风俗 |
| 评论展示 | 按时间/热度排序展示评论 | 基础需求 |

### 2.5 成就系统（预留功能）

| 功能模块 | 功能描述 | 需求来源 |
| :--- | :--- | :--- |
| 成就徽章 | 集满一个城市的特点后解锁成就徽章 | 用户要求：成就系统 |
| 成就展示 | 在个人中心展示已获得的徽章 | 预留功能 |

### 2.6 分享功能

| 功能模块 | 功能描述 | 需求来源 |
| :--- | :--- | :--- |
| 打卡分享 | 将打卡内容分享到社交平台 | 用户要求：分享功能 |
| 评论分享 | 分享精彩评论到社交平台 | 基础需求 |

## 4. 用户流程

### 4.1 注册登录流程
```
用户打开App → 选择注册/登录 → 输入账号密码 → 完成验证 → 进入首页
```

### 4.2 城市打卡流程
```
首页 → 选择城市 → 查看城市详情 → 选择打卡类型（景点/美食/风俗）→ 上传图片 → 填写描述 → 发布打卡
```

### 4.3 评论打分流程
```
城市详情 → 选择评论对象 → 输入评论内容 → 选择评分（1-5星）→ 发布评论
```

## 6. 非功能需求

### 6.1 性能需求
- 页面加载时间 ≤ 2秒
- 图片加载优化，支持懒加载
- 接口响应时间 ≤ 500ms

### 6.2 兼容性需求
- 支持 iOS 12+ 和 Android 6.0+
- 支持主流屏幕尺寸

### 6.3 安全性需求
- 用户密码加密存储（MD5/SHA256）
- 接口请求需携带token验证
- 图片上传需进行安全检测
