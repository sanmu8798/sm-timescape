# 【时光集】产品需求文档（PRD）

## 1. 产品概述

### 1.1 产品名称
**时光集** - 记录旅行足迹，收藏美好时光

### 1.2 产品定位
旅行记录与发现工具，帮助用户记录全国各个城市的景点、美食、风俗特色，通过打卡解锁成就徽章、发布动态、评论打分等功能，打造个人专属的旅行记忆集。

### 1.3 目标用户
- **旅行爱好者**：经常旅游，喜欢记录旅行点滴
- **本地居民**：分享本地特色，展示家乡魅力
- **学生群体**：预算有限的年轻旅行者
- **商务人士**：出差期间探索当地美食

### 1.4 产品愿景
成为用户记录旅行、分享体验、发现美好的首选平台，让每一次旅行都留下珍贵印记。

## 4. 用户流程

### 4.1 登录流程
```
用户打开 App → Demo 阶段自动登录 → 进入首页
```

### 4.2 城市打卡流程
```
首页/发现 → 选择城市/景点/美食/风俗 → 查看详情 → 点击打卡 → 上传图片/填写描述 → 发布打卡 → 徽章进度更新 → 用户统计更新 → 省份点亮更新
```

### 4.3 评论打分流程
```
城市/景点/美食/风俗详情 → 查看评价 → 进入评论列表 → 选择评分（1-5星）→ 输入评论内容 → 发布评论 → 评论列表更新
```

### 4.4 徽章解锁流程
```
完成城市内景点/美食/风俗打卡 → 城市徽章进度累计 → 进度达到 100% → 自动解锁城市徽章
完成特殊目标（如打卡八达岭长城）→ 解锁对应特色徽章
```

### 4.5 动态广场流程
```
进入广场页 → 浏览推荐/关注/热门动态流 → 点赞/评论/收藏/分享动态 → 点击城市标签进入城市详情
```

### 4.6 个人中心流程
```
进入我的页面 → 查看用户资料与数据看板 → 点击点亮进度/成就徽章/足迹时间线 → 进入对应页面
```

## 6. 技术实现要点

### 6.1 前端技术栈
- Vue 3 + `<script setup>`
- Vue Router 4
- Pinia 状态管理
- Vant UI 组件库
- Less 预处理器 + CSS 变量
- lucide-vue-next 图标库

### 6.2 Mock 数据规范
- 统一存放在 `src/mock/` 目录
- 关键列表数据使用 Vue `reactive()` 包装，支持响应式更新
- 严格保持数据关联一致性：`city_id`、`target_id`、`target_type`、`target_name` 必须与对应数据表匹配
- 历史故事使用富文本 HTML，包含标题、段落、引用、图文并茂的 figure

### 6.3 数据联动规则
- 新增打卡 → 写入 `checkins` → 调用 `refreshBadges()` 更新徽章进度 → 调用 `syncUserStats()` 同步用户统计与省份点亮 → 刷新 Pinia 用户数据
- 新增评论 → 写入 `comments` → 评论列表自动更新

### 6.4 响应式数据管理
- `checkins`、`comments`、`badges`、`provinces` 等核心列表使用 Vue `reactive()` 包装
- 页面通过 `computed()` 读取派生数据，保证新增/删除/状态变更后 UI 自动刷新

### 6.5 页面结构与样式规范
- 页面文件统一使用 `<template>` + `<script setup>` + `<style lang="less" scoped>` 三段式结构
- 样式优先使用 CSS 变量，禁止在页面内重复定义全局 `.rich-text` 等公共样式
- 图标统一使用 `lucide-vue-next`，禁止自定义 SVG

## 7. 非功能需求

### 7.1 性能需求
- 页面加载时间 ≤ 2 秒
- 图片加载优化，支持懒加载
- 接口响应时间 ≤ 500ms

### 7.2 兼容性需求
- 支持 iOS 12+ 和 Android 6.0+
- 支持主流屏幕尺寸，移动端优先

### 7.3 安全性需求
- 用户密码加密存储
- 接口请求需携带 token 验证
- 图片上传需进行安全检测

### 7.4 交互与反馈需求
- 关键操作（发布打卡、发布评论、点赞）需给出 Toast 提示
- 分享功能 Demo 阶段使用"即将上线"提示，不调用真实分享 SDK
- 页面切换使用 Vue Router 命名路由，返回统一使用 `router.back()`
- 图片使用本地资源模拟上传，多张图片以网格形式展示
