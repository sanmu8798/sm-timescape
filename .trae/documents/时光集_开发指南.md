# 【时光集】开发指南

## 1. 技术栈选择

### 1.1 前端框架
- **框架**: Vue 3 + TypeScript
- **UI框架**: Element Plus (Vue 3 版本)
- **路由**: Vue Router 4
- **状态管理**: Pinia
- **HTTP请求**: Axios
- **图标库**: Element Plus Icons / Lucide Icons

### 1.2 后端技术
- **框架**: Node.js + Express / NestJS
- **数据库**: MongoDB (文档型数据库，适合存储打卡、评论等非结构化数据)
- **文件存储**: 阿里云OSS / 腾讯云COS
- **认证**: JWT Token

### 1.3 开发工具
- **构建工具**: Vite
- **代码检查**: ESLint + Prettier
- **包管理**: npm / yarn
- **版本控制**: Git

## 3. 核心功能开发说明

### 3.1 用户系统

#### 3.1.1 注册登录
- **API接口**:
  - `POST /api/user/register` - 用户注册
  - `POST /api/user/login` - 用户登录
- **流程**:
  1. 用户输入账号密码
  2. 调用登录接口获取token
  3. 将token存储到localStorage
  4. 更新用户状态

#### 3.1.2 用户状态管理
- 使用Pinia管理用户状态
- 包含用户信息、token、登录状态

### 3.2 城市探索

#### 3.2.1 城市列表
- **API接口**:
  - `GET /api/city` - 获取城市列表
  - `GET /api/city?keyword=xxx` - 搜索城市
- **功能**:
  - 展示全国城市列表
  - 支持搜索筛选
  - 展示城市封面图和简介

#### 3.2.2 城市详情
- **API接口**:
  - `GET /api/city/:id` - 获取城市详情
  - `GET /api/city/:id/attractions` - 获取城市景点
  - `GET /api/city/:id/foods` - 获取城市美食
  - `GET /api/city/:id/customs` - 获取城市风俗

### 3.3 打卡系统（核心功能）

#### 3.3.1 打卡发布
- **API接口**:
  - `POST /api/checkin` - 发布打卡
- **功能**:
  - 选择打卡类型（景点/美食/风俗）
  - 上传图片（最多9张）
  - 填写打卡描述
  - 获取当前位置

#### 3.3.2 打卡记录
- **API接口**:
  - `GET /api/checkin/user/:userId` - 获取用户打卡记录
- **功能**:
  - 按时间顺序展示打卡记录
  - 支持图片瀑布流展示

### 3.4 评论评分系统

#### 3.4.1 发布评论
- **API接口**:
  - `POST /api/comment` - 发布评论
- **功能**:
  - 输入评论内容
  - 选择评分（1-5星）
  - 支持对城市/景点/美食/风俗评论

#### 3.4.2 评论列表
- **API接口**:
  - `GET /api/comment/:type/:targetId` - 获取评论列表
- **功能**:
  - 按时间/热度排序展示
  - 显示用户头像和昵称

## 4. API接口设计

### 4.1 用户接口

| 方法 | 路径 | 描述 |
| :--- | :--- | :--- |
| POST | /api/user/register | 用户注册 |
| POST | /api/user/login | 用户登录 |
| GET | /api/user/profile | 获取用户信息 |
| PUT | /api/user/profile | 更新用户信息 |

#### 请求示例

**注册**:
```json
{
  "username": "string",
  "phone": "string",
  "password": "string",
  "email": "string (可选)"
}
```

**登录**:
```json
{
  "phone": "string",
  "password": "string"
}
```

### 4.2 城市接口

| 方法 | 路径 | 描述 |
| :--- | :--- | :--- |
| GET | /api/city | 获取城市列表 |
| GET | /api/city/:id | 获取城市详情 |
| GET | /api/city/:id/attractions | 获取城市景点 |
| GET | /api/city/:id/foods | 获取城市美食 |
| GET | /api/city/:id/customs | 获取城市风俗 |

### 4.3 打卡接口

| 方法 | 路径 | 描述 |
| :--- | :--- | :--- |
| POST | /api/checkin | 发布打卡 |
| GET | /api/checkin/user/:userId | 获取用户打卡记录 |
| GET | /api/checkin/:id | 获取打卡详情 |
| DELETE | /api/checkin/:id | 删除打卡 |

#### 请求示例

**发布打卡**:
```json
{
  "type": "attraction/food/custom",
  "targetId": "string",
  "images": ["string"],
  "content": "string",
  "location": "string"
}
```

### 4.4 评论接口

| 方法 | 路径 | 描述 |
| :--- | :--- | :--- |
| POST | /api/comment | 发布评论 |
| GET | /api/comment/:type/:targetId | 获取评论列表 |
| DELETE | /api/comment/:id | 删除评论 |

#### 请求示例

**发布评论**:
```json
{
  "type": "city/attraction/food/custom",
  "targetId": "string",
  "content": "string",
  "rating": 1-5
}
```

## 6. 开发流程

### 6.1 环境搭建
1. 安装 Node.js（建议 v18+）
2. 安装 MongoDB
3. 克隆项目代码
4. 安装依赖：`npm install`
5. 启动开发服务器：`npm run dev`

### 6.2 代码规范
- 使用 ESLint + Prettier 进行代码检查
- 组件命名使用 PascalCase
- 文件命名使用 kebab-case
- TypeScript 类型定义必须完整

### 6.3 分支管理
- `main`：主分支，稳定版本
- `develop`：开发分支
- `feature/*`：功能开发分支
- `bugfix/*`：bug修复分支

### 6.4 提交规范
```
feat: 添加新功能
fix: 修复bug
docs: 更新文档
style: 代码格式调整
refactor: 代码重构
test: 添加测试
```

## 7. 部署说明

### 7.1 前端部署
- 构建生产版本：`npm run build`
- 部署到静态服务器（Nginx / 阿里云OSS / Netlify）

### 7.2 后端部署
- 使用 PM2 管理 Node.js 进程
- 配置 Nginx 反向代理
- 配置 SSL 证书

### 7.3 数据库部署
- 使用 MongoDB Atlas 云数据库
- 或自建 MongoDB 集群

---

## 8. 注意事项

1. **图片上传**: 使用云存储服务，避免直接存储在服务器
2. **位置权限**: 需要用户授权获取位置信息
3. **Token过期处理**: 统一处理token过期，自动跳转登录页
4. **图片懒加载**: 使用 Intersection Observer API 实现
5. **响应式设计**: 适配移动端和PC端
